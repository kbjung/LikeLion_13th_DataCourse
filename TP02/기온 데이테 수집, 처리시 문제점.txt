### 2013~2018년 시간별 기온 데이터 수집

1. **기온 데이터 크롤링 과정 중 겪은 문제점**
- 달마다 날짜가 다른 문제
    - 매년 2월 28일까지 존재, 2016년 2월은 29일까지 존재
    - 30, 31일
        
         ⇒ 특정 연, 월, 일 데이터 크롤링하는 함수 작성하여 조건문으로 해결.
        
- 농업기상정보시스템 데이터베이스에서 결측치 존재
    - 결측치가 존재하는 날짜, 시간이 존재했다.
        
        ⇒ 해당 날짜의 시간별 리스트 만들어 대조해서 확인. 날짜가 없으면 해당 날짜 채우고, 기온이 없을 경우 NaN값으로 처리.
        
- 크롤링시 bs 리스트에서 날짜, 기온의 데이터 인덱스 순서가 달라지는 문제.
    - 2014년 6월12일까지 13번째마다 날짜, 기온 데이터 반복.
    - 이후부터 15번째마다 반복.
        
        ⇒ 각각 함수화해서 조건마다 적용.
        
- 중복 날짜 데이터가 수집되는 문제
    
    ⇒ 날짜 데이터를 set으로 변환해 중복 제거.
    
- 가스공급량, 기온 데이터 셋에서 날짜 데이터가 배열 순서 다른 문제
    - 가스공급량에서는 오름차순 배열, 기온 데이터 셋에서는 내림차순 배열 .
        
        ⇒ 기온 데이터 셋에서 날짜 데이터를 list로 변경해 오름차순 정렬.
        
- 가스 공급량과 기온 데이터 셋의 날짜 표기 방식 차이 문제.
    - 기온데이터에서 해당 날짜의 24시가 다음날 0시로 표현되는 문제.
        
        ⇒ 날짜의 마지막 0시를 하루 당겨 24시로 변경.
        
1. **데이터 전처리 과정 중 겪은 문제점**
- 특정 날짜 오후, 오전 시간 때 기온 데이터 없는 문제.
    
    ⇒ 다른 해, 같은 달, 일, 시간의 기온 평균으로 결측치 처리.
    
- 매년 12월31일 24시 기온 데이터 없는 문제.
    
    ⇒ 연도별 바로 전시간 23시 기온으로 결측치 처리.
